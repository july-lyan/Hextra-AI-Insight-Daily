### Product & Feature Updates

1.  **Claude Desktop now supports local Claude Code.**
    For developers not keen on command-line terminals, this is a game-changer! You can now use Claude Code locally directly within [Claude Desktop (AI News)](https://claude.ai/download) with its graphical interface. Just toggle the "Code" switch in the sidebar, authorize a folder, and let Claude help you write code and modify projects. It also supports **parallel multi-tasking**, allowing multiple sessions without interference, just like developing two branches simultaneously. Enterprise users also benefit from permission control, ensuring security and peace of mind.<br/>![AI News: Claude Desktop Code Feature Switch Interface](https://source.hubtoday.app/images/2026/01/news_01kechshz0f1at9a6fahbmve5f.avif)<br/>

2.  **Cursor launches Dynamic Context Discovery Mode.**
    Cursor has launched its Dynamic Context Discovery Mode. While traditional AI assistants often cram all information into the prompt box, leading to skyrocketing token consumption, Cursor takes the opposite approach. Its Agent dynamically fetches context only when needed, keeping prompts concise. The [Official Blog (AI News)](https://cursor.com/blog/dynamic-context-discovery) reports that using MCP tools can **reduce token consumption by an average of 46.9%**. Got a super long chat history? Just convert it into a file and let the Agent search, so you'll never miss a detail again!<br/>![AI News: Cursor Dynamic Context Discovery Architecture Diagram](https://source.hubtoday.app/images/2026/01/news_01kechsqn8fjrsmnrxyza86ehp.avif)<br/>

    Liquid AI has released its LFM2.5 family of small models for edge devices. ### Cutting-Edge Research

1.  **VTONQA: First virtual try-on quality assessment dataset released.**
    VTONQA, the first virtual try-on quality assessment dataset, has been released. Virtual try-on images for online shopping often get distorted or look wonky, but now there's a dedicated evaluation standard. The [arXiv Paper (AI News)](https://arxiv.org/abs/2601.02945) reveals that the dataset contains **8,132 images** generated by 11 try-on models, with scores across three dimensions: garment fit, body compatibility, and overall quality. This release highlights the limitations of existing models and provides clear direction for developers.

2.  **CodeEvolve: Open-source evolutionary code generation framework.**
    CodeEvolve, an open-source evolutionary code generation framework, has been launched. It combines large models with evolutionary search to synthesize high-performance algorithms. The [GitHub Open Source (AI News)](https://github.com/inter-co/science-codeevolve) repository shows it outperforms AlphaEvolve in benchmark tests. It's pretty wild that an open-source model can rival closed-source solutions, and with **just a fraction of the computational cost**!
